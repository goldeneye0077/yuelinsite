# Phase 3 UI Spec

## Visual Thesis

An engineering catalog index rather than a storefront grid: product taxonomy should read like structured documentation with clear lines, counts, and grouped series names.

## Composition Rules

- Keep the product-center preview list-led and card-light.
- Use one side for primary family framing and one side for the deeper industrial-sensor subgroup scan.
- Count labels and source attribution should feel factual, not decorative.
- The preview should feel like a taxonomy scaffold, not a fake finished catalog.

## Motion Rules

- Reuse the restrained rise-in motion already established in Phase 2.
- Do not add extra kinetic effects just because the page is more information-dense.
