# Plan 05-03

## Objective

Complete the live-browser QA pass, remove delivery blockers found in Compose/browser usage, and harden the site so responsive browsing, theme behavior, and inquiry submission all hold up in a handoff environment.

## Acceptance Criteria

- Core desktop and mobile routes browse without blocked navigation or obvious layout breakage
- The inquiry form can submit successfully from the browser against the Compose backend
- Compose startup works even when the site is exposed on alternate host ports
- Theme controls and global navigation feel stable and readable during QA
- Containerized frontend test/build commands can run against the shared node-modules volume without flaky install races
- Frontend dependency audit no longer reports the previously known critical vulnerabilities

## Verification

- `pytest -q`
- `npm run typecheck`
- `npm run lint`
- `npm run test -- --run`
- `npm run build`
- `npm audit --omit=dev`
- `docker compose config`
- `docker compose up -d`
- Browser smoke test across `/zh`, `/en`, `/zh/products`, `/zh/solutions`, `/zh/support`, `/zh/about`, and `/zh/contact`
