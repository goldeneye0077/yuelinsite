# Plan 05-01

## Objective

Build the first real inquiry loop by replacing the placeholder contact route with a dedicated contact page, then wiring the backend to validate, persist, and confirm inquiry submissions.

## Acceptance Criteria

- Users can open a dedicated bilingual contact page with company coordinates, inquiry framing, and a visible form
- Users can submit essential inquiry fields and receive clear success or failure feedback on the page
- The backend validates inquiry payloads and persists accepted submissions to the database
- Local and Compose startup paths can bring up the API with the required inquiry table available
